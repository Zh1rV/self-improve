# Checkpoint Template

Use this format for each iteration checkpoint.

## Iteration <N>

- Objective:
- Files changed:
- Validation commands:
- Validation outcomes:
- Residual risk:
- Stop reason (if stopping):

Notes:

- When present, `Stop reason` should use a code from `references/stop-reasons.md`.
- `Validation commands` should include the exact command text, not just a summary.

## Final Summary

- Completed improvements (ordered):
- Validation evidence summary:
- Deferred opportunities:
- Recommended next actions:
